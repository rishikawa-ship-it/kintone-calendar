# kintone フィールド参照一覧

## 1. フィールド定義（KC.Config.FIELD）

| 内部キー | フィールドコード | kintone型 | 必須 | 用途 |
|---|---|---|---|---|
| `title` | 予定タイトル | SINGLE_LINE_TEXT | **必須** | イベントタイトル表示・入力 |
| `status` | 貸出ステータス | DROP_DOWN | 任意 | クエリ除外フィルタ・ダイアログ選択 |
| `start` | 開始日時 | **DATETIME または DATE** | **必須** | 期間フィルタ・イベント配置・ソート |
| `end` | 終了日時 | **DATETIME または DATE** | **必須** | 期間フィルタ・イベント配置 |
| `allday` | 終日 | CHECK_BOX | 任意 | 終日/通常の表示分岐（DATETIME型のときのみ参照） |
| `place` | 場所 | SINGLE_LINE_TEXT | 任意 | ダイアログ入力のみ |
| `userName` | 利用者氏名 | SINGLE_LINE_TEXT | 任意 | ダイアログ（ログインユーザー自動セット） |
| `userMail` | 利用者メールアドレス | SINGLE_LINE_TEXT | 任意 | ダイアログ（ログインユーザー自動セット） |
| `account` | アカウント | USER_SELECT | 任意 | ログインユーザー自動セット（UI非表示） |
| `memo` | 説明欄 | MULTI_LINE_TEXT | 任意 | ダイアログ入力のみ |

※ システムフィールド `$id`, `$revision`, `$created` もAPI取得時に使用

### start / end フィールド型の補足

`start` と `end` には **DATETIME 型または DATE 型** を使用できる。フィールド型は kintone アプリ設計時に決定し、本カスタマイズは起動時の `KC.Config.detectFields()` でフィールド型を自動検出して `KC.Config.START_FIELD_TYPE` に格納する（`'DATETIME'` または `'DATE'`）。

**DATE 型を選択した場合**: kintone の DATE 型フィールドは時間情報を持たない。そのため、`start` / `end` に DATE 型を使用しているアプリのイベントは **自動的に終日扱い** となり、`allday` フィールドの値は無視される。時間ありレーンに描画する選択肢は物理的に存在しないためである。

この設計思想により、日にち単位でレコードを管理するアプリ（時間を扱わない業務）でも本カレンダーカスタマイズを採用できる。

### フィールド型の選択指針

| ユースケース | 推奨フィールド型 | allday フィールド |
|---|---|---|
| 時間あり予定と終日予定を混在させたい | DATETIME | 必要（CHECK_BOX） |
| すべての予定を終日（日付）で管理したい | DATE | 不要（省略可） |
| 日にち単位の業務レコード（例: 機器貸出日） | DATE | 不要（省略可） |

## 2. ステータスの固定文字列

### ダイアログ選択肢（ハードコード）
- `""` （未設定）
- `"予約済"`
- `"貸出中"`
- `"返却済"`

### クエリ除外条件（ハードコード）
- `"返却済"` — カレンダー非表示
- `"削除済"` — カレンダー非表示（UIからは選択不可）

### 不整合
`EXCLUDED_STATUSES` 定数が定義されているが、クエリ構築では直接文字列リテラルが使用されている

## 3. 汎用化時の変更箇所

### A. 必須変更（フィールドコード依存）

| 箇所 | 行番号付近 | 内容 |
|---|---|---|
| `KC.Config.FIELD` | 21-31行 | フィールドコード定義 → 外部設定化 |
| `KC.Config.START_FIELD_TYPE` | 55行 | `'DATETIME'` または `'DATE'` を `detectFields` で自動設定済み（汎用化対応済み） |
| ステータス `<select>` | 259-263行 | 選択肢ハードコード → kintone APIで動的取得 |
| クエリ除外条件 | 143行 | 文字列直書き → `EXCLUDED_STATUSES` 参照に修正 |
| チェックボックス値 `'終日'` | 129, 167, 189行 | 設定値化 |

> `KC.Config.START_FIELD_TYPE` は `KC.Config.detectFields()` によって実行時に自動検出され上書きされる。DATE 型の場合の終日判定・クエリ形式変換・保存時の値フォーマット変換はすべてこの値を参照して分岐する。

### B. UI文言

| 箇所 | 内容 |
|---|---|
| ヘッダータイトル | `'貸与Wi-Fi予約カレンダー'` |
| ダイアログラベル | `予定タイトル`, `機器名` 等 |
| バリデーションメッセージ | 日本語固定 |

### C. ビジネスロジック

| 箇所 | 内容 |
|---|---|
| ログインユーザー自動セット | userName/userMail/account への自動入力 |
| ダイアログのフォーム構造 | フィールド構成がハードコード |

## 4. 汎用化の推奨アプローチ

1. `KC.Config` を外部から注入可能にする（プラグイン設定画面 or 設定用JSファイル）
2. ダイアログのフォームを `FIELD` 定義から動的生成
3. ステータス選択肢をkintone APIのフィールド定義から取得
4. `EXCLUDED_STATUSES` を設定経由で参照
5. UI文言をラベル定義オブジェクトに集約
