## §1 背景・目的

### 1.1 背景

KC Calendar プラグインの設定画面には「権限フィールド設定」と「権限ユーザー設定」の 2 つの権限テーブルがある（REQ_edit-permission-extension v6 で実装済み）。

`desktop.js` の `getPermission` 関数は、各テーブルの行を**配列の先頭から順番に走査**し、最初にマッチしたルールを採用する仕様である（§5.10 擬似コード参照）。ただし現在の設定画面 UI にはルールの上下順を変更する手段がなく、管理者は一度追加した行の優先順序を変更できない。

ユーザー要望「権限テーブルの上下を変えられるようにしてください。（上の権限が優先されるようにしてください。）」を受け、設定 UI に並び替え機能を追加する。

### 1.2 目的

1. 設定画面の権限テーブルに行の上下並び替え操作を提供し、管理者が優先順序を直感的に制御できるようにする
2. 評価ロジックにおいて「上の行ほど優先される」ことを UI・ドキュメント上で明示し、誤解を防ぐ
3. 並び順を設定値として保存・復元できるようにする（配列の格納順が優先順序と一致する既存の保存形式を利用）

## §2 現状分析

### 2.1 対象テーブルの概要

`plugin/src/html/config.html` に 2 つの権限テーブルが存在する。

| テーブル | コンテナ ID | 行クラス | 行追加ボタン ID |
|---|---|---|---|
| 権限フィールド設定 | `kc-fieldvalue-rows` | `.kc-fieldvalue-row` | `kc-fieldvalue-add` |
| 権限ユーザー設定 | `kc-permission-rows` | `.kc-permission-row` | `kc-permission-add` |

(`config.html:257`, `config.html:288`)

### 2.2 行の生成・削除

- **権限フィールド設定**: `buildFieldValueRow(rule)` が 1 行の `<div class="kc-fieldvalue-row">` を生成し、`elFieldValueRows.appendChild()` でコンテナへ追加する (`config.js:1143`)
- **権限ユーザー設定**: `buildPermissionRow(rule)` が 1 行の `<div class="kc-permission-row">` を生成し、`elPermissionRows.appendChild()` で追加する (`config.js:900`)
- **削除**: 各行内の「−」ボタンがクリックされると `row.parentNode.removeChild(row)` で DOM から直接除去する (`config.js:923`, `config.js:1160`)
- **行識別 key**: 各行に id 属性や専用の識別子はなく、DOM の順番（index）でのみ管理されている

### 2.3 設定の収集・保存形式

- `collectPermissionRules()`: `elPermissionRows.querySelectorAll('.kc-permission-row')` で取得した NodeList を先頭から順に処理し `permissionRules` 配列を返す (`config.js:1033`)
- `collectFieldValueRules()`: 同様に `elFieldValueRows.querySelectorAll('.kc-fieldvalue-row')` で `fieldValueRules` 配列を返す (`config.js:1313`)
- 保存形式は JSON 配列。**配列の格納順が直接 UI の表示順・評価順に対応する**。優先度を示す別フィールドは存在しない

### 2.4 評価ロジックにおける評価順序

`desktop.js` の `getPermission` 関数（`desktop.js:2426`）:

1. `KC.Config.FIELDVALUE_RULES` 配列を **先頭から順番に** `for` ループで走査し、マッチした最初の行を採用して即 `return` する (`desktop.js:2431`)
2. フィールド値ルールにマッチしなかった場合、`KC.Config.PERMISSION_RULES` を走査する（権限は OR 結合で最高権限採用、色は先頭マッチ優先）(`desktop.js:2449`)

すなわち「配列の上（先頭）の行ほど優先される」ロジックはすでに **desktop.js 側で実装済み**である。ただし、設定画面 UI に並び替え操作が存在しないため、この評価順序を管理者が制御する手段がない。

### 2.5 現状の制約

- 行の追加は末尾追加のみ（`appendChild`）
- 行の順番を変更する UI（ボタン / DnD / 数値入力など）は未実装
- 行の key 管理は DOM index のみ（並び替えのための id 等は不要）

## §3 機能要件

### FR-1: HTML5 DnD による行の並び替え

権限テーブルの各行に、HTML5 Drag-and-Drop API を用いた並び替え機能を実装する。

**実装方式: HTML5 DnD（確定）**

- 各行要素（`.kc-fieldvalue-row` / `.kc-permission-row`）の左端にドラッグハンドルアイコン（「⋮⋮」グリップアイコン推奨）を配置する
- ハンドル要素に `draggable="true"` を付与し、行全体ではなくハンドル部分からのみドラッグを開始する
- 実装する HTML5 DnD イベント: `dragstart`, `dragover`, `drop`, `dragend`, `dragenter`, `dragleave`
- ドロップ時は `insertBefore` / `appendChild` で DOM 順序を変更する

**行内要素との競合回避**

- ドラッグ可能領域は専用ハンドルアイコン（または行の左端余白部分）に限定し、行全体を `draggable="true"` にしない
- これにより、行内の `<input>`, `<select>`, `<button>` 要素（カラーピッカー・権限選択・削除ボタン等）への通常クリック・入力操作と競合しない

**ドラッグ中の視覚フィードバック**

- ドラッグ元行の不透明度を下げる（例: `opacity: 0.4`）
- カーソルをドラッグ中は `grabbing` に変化させる（ハンドル部分は通常時 `grab`）
- ドロップ可能な挿入位置にハイライト線（水平ボーダー等）を表示する（`dragenter` / `dragleave` イベントで制御）

**iframe 環境での挙動**

- kintone プラグイン設定画面は iframe 内で動作する
- iframe 外からの `dragenter` イベントは無視する（別ウィンドウ / 別タブからのドラッグは対象外）
- iframe 内の同一テーブル内でのドラッグ & ドロップのみを有効とする

**ハンドルの推奨配置**

- 各行の左端（既存の削除ボタン「−」の横ではなく、行の先頭）に配置する
- アイコン例: 「⋮⋮」（vertical dots / grip lines）

### FR-2: 「上の行ほど優先」評価ロジックの明示

`desktop.js` の `getPermission` 関数（`desktop.js:2426`）はすでに「配列の先頭から順に走査し、最初にマッチしたルールを採用」する実装となっており、コードの変更は不要である。

本要件での対応は以下のみ:

- config.html の権限テーブル付近に「上の行ほど優先されます」という旨のヒントテキストを追加する
- 「権限ユーザー設定」については「権限は複数マッチ時に最高権限が適用されます。色は上の行のマッチが優先されます」と明示する

### FR-3: 並び順の保存・復元

- 保存時: 並び替え後の DOM 行の順序どおりに `collectPermissionRules()` / `collectFieldValueRules()` が配列を収集し、その順序のまま `kintone.plugin.app.setConfig` に保存する
- 復元時: `applyPermissionRules(rules)` / `applyFieldValueRules(rules)` が配列の順序どおりに DOM 行を追加することで、リロード後も並び順が維持される
- **config スキーマの変更は不要**。既存の配列順＝評価順の設計をそのまま利用する

### FR-4: 適用範囲（確定）

並び替え機能を「権限フィールド設定」「権限ユーザー設定」の**両テーブル**に適用する（確定）。

- `kc-fieldvalue-rows` コンテナ内の `.kc-fieldvalue-row` 行
- `kc-permission-rows` コンテナ内の `.kc-permission-row` 行

いずれも同一の DnD 実装パターンを適用する。

## §4 受入基準

### AC-1: DnD による行の移動操作が可能

**Given** 権限テーブルに 2 行以上のルールが存在する
**When** 管理者が任意の行のドラッグハンドルをつかみ、別の行の位置へドラッグしてドロップする
**Then** ドロップした位置にその行が挿入され、他の行が相対的に再配置される

**Given** ドラッグ操作中
**When** ドラッグ元行がドラッグされている
**Then** ドラッグ元行の不透明度が下がり（視覚的に「移動中」を示す）、ドロップ可能位置にハイライト線が表示される

**Given** ドラッグ操作が完了する（drop または dragend）
**When** ドロップが成功した場合
**Then** ハイライト線・不透明度の変化が元に戻り、新しい並び順で行が表示される

**Given** ドラッグ操作中に Escape キーを押すか、テーブル外にドロップする
**When** キャンセルされる
**Then** 行の並び順は変化せず元に戻る（dragend イベントで復元）

### AC-2: 行内要素との操作競合がない

**Given** 各行に `<input>`, `<select>`, カラーピッカーボタン、削除ボタンが存在する
**When** 管理者がこれらの要素をクリック・入力する
**Then** DnD が誤って発動せず、通常の入力・クリック操作が行われる

**Given** ドラッグハンドルアイコン以外の行内領域をドラッグしようとする
**When** 行内テキスト・入力フィールド等を誤ってドラッグする
**Then** テキスト選択などブラウザ既定の動作となり、行の DnD は発動しない

### AC-3: 並び順が保存・復元される

**Given** 管理者が権限テーブルの行を DnD で並び替えた後、「保存」または「保存して更新」ボタンを押す
**When** 設定画面をリロードする
**Then** 並び替え後の順序で行が表示される

**Given** `kintone.plugin.app.setConfig` に保存された `permissionRules` または `fieldValueRules` の配列を確認する
**Then** 設定画面で表示されている行の順序と配列の格納順が一致している

### AC-4: 「上が優先」の評価ロジックが動作する

**Given** 「権限フィールド設定」テーブルに 2 つのルールが存在し、同一レコードに両方のルールが一致する
**When** カレンダー画面でそのレコードが描画される
**Then** テーブルの上（先頭）にあるルールの `permission` / `bgColor` / `textColor` が適用される（下のルールは評価されない）

**Given** 上記ルールを DnD で並び替えて下のルールを上に移動し、設定を保存する
**When** カレンダー画面でそのレコードが描画される
**Then** 移動後に上になったルールが適用される

### AC-5: 行内の設定値が移動後も保持される

**Given** フィールドコード・権限・背景色・文字色が設定された行を DnD で移動する
**When** ドロップ操作が完了する
**Then** 移動後の行に設定値（フィールドコード・権限・背景色・文字色）が正しく引き継がれている

## §6 設計（採用案: HTML5 DnD）

### 採用案: HTML5 Drag-and-Drop（案 B）— 確定

```
[⋮⋮ハンドル] [フィールド] [値] [権限] [背景] [文字] [−]
```

**採用理由**: ユーザーが HTML5 DnD を明示的に選択。

#### 実装仕様

**行レイアウト変更**

- 既存の各行（`.kc-fieldvalue-row` / `.kc-permission-row`）の先頭にドラッグハンドル要素を追加する
- ハンドル要素例: `<span class="kc-drag-handle" title="ドラッグして並び替え">⋮⋮</span>`
- ハンドル要素に `draggable="true"` を付与する（行全体には付与しない）

**DnD イベント実装**

| イベント | 付与先 | 処理内容 |
|---|---|---|
| `dragstart` | ハンドル要素 | ドラッグ対象行を変数に保持。行の不透明度を下げる |
| `dragover` | 各行要素 | `preventDefault()` を呼んでドロップを許可。挿入位置のハイライト線を表示 |
| `dragenter` | 各行要素 | ドロップターゲット行にハイライトクラスを付与 |
| `dragleave` | 各行要素 | ハイライトクラスを除去 |
| `drop` | 各行要素 | `insertBefore` で行を挿入。ハイライトクラスを除去 |
| `dragend` | ハンドル要素 | 不透明度を元に戻す。全ハイライトを除去（キャンセル時の安全処理） |

**視覚フィードバック詳細**

- ドラッグ元行: `opacity: 0.4`（dragstart で付与、dragend で除去）
- ハンドルカーソル: 通常時 `cursor: grab`、ドラッグ中 `cursor: grabbing`
- ドロップ位置ハイライト: ターゲット行の上端または下端に水平ボーダー線を表示（CSS クラスで制御）

**iframe 対応**

- `dragover` イベントで `dataTransfer.types` を確認し、外部ファイルドロップ等の不要なドラッグを無視する
- テーブルコンテナ外へのドロップは `dragend` の `dataTransfer.dropEffect === 'none'` で検出し、DOM を復元する

### 前提・リスク

- 評価ロジック（`desktop.js:2426` の `getPermission`）は**現状すでに配列先頭優先**で実装されており、本機能追加に際してロジック変更は不要
- `config.js` の `collectPermissionRules()` / `collectFieldValueRules()` は DOM 順で配列を収集するため、DOM 順を変更するだけで保存順序が変わる（追加実装が最小限で済む）
- 行 key 管理は DOM index のみであり、並び替えのための id 付与は不要（DOM 操作で完結）
- kintone.plugin.app.setConfig の 65535 バイト制限に変化はない（並び替えはバイト数を変えない）

## 付録: UX / シーケンス

### 設定画面の操作フロー（DnD 並び替え追加後）

```
1. 管理者がプラグイン設定画面を開く（kintone iframe 内）

【権限フィールド設定 / 権限ユーザー設定】
2. テーブルの各行左端に「⋮⋮」ドラッグハンドルアイコンが表示される
3. ハンドルにカーソルを合わせると cursor: grab に変化する
4. 管理者がハンドルをドラッグ開始（dragstart）:
   - ドラッグ元行の opacity が 0.4 に下がる
5. 他の行上にドラッグ（dragover / dragenter）:
   - ドロップ挿入位置にハイライト線が表示される
6. ドロップ（drop）:
   - 行が挿入位置に移動（insertBefore で DOM を更新）
   - ハイライト・opacity が元に戻る（dragend）
7. キャンセル時（Escape / テーブル外ドロップ）:
   - dragend で DOM を復元し、並び順は変化しない

8. 「保存」または「保存して更新」ボタンを押す
9. collectPermissionRules() / collectFieldValueRules() が DOM の表示順で配列を収集
10. setConfig に保存される配列の順序 = DnD 後の並び順

11. カレンダービューで評価時（desktop.js:getPermission）:
    - fieldValueRules[0] から順に走査 → 最初マッチを採用
    - 上の行ほど優先されることをヒントテキストで周知済み
```
